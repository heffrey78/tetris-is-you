Multi-Line Clearing Bug Fix Test

üêõ Bug Description

Issue: Multi-line clearing was not working correctly. When two or more lines should be cleared simultaneously, the bottom line would not clear properly due to array index shifting issues.

Root Cause: The line removal logic was flawed - it was removing lines one by one while adding new empty lines after each removal, causing array index shifting. When clearing lines [18, 19], after removing line 18, the original line 19 became line 18, but the code still tried to remove index 19.

üîß Fix Applied

Fixed the multi-line clearing synchronization issue with two key changes:

Changes made in GameLogic.ts:
1. Fixed line removal logic: Separated line removal from empty line addition
- Remove all completed lines first (bottom to top to avoid index shifting)
- Then add all new empty lines at the top in a separate loop
2. Added debugging: Enhanced console logging to track the clearing process
3. Existing chain reaction prevention: The isClearing flag prevents spell chain reactions during line clearing
